Prompt 1:

Using C#:
We want to create a graphical labeling tool to assist a human operator to find the coordinates of the drone in the image (if any), outputting a single txt files in YOLO format for each entry.
We wish to classify if our pairs of images contain drones or not, with coordinates of the drones in the images.
The files are pairs of grayscale PNG images, with paths
"dataset\grey\clear\<id>_clear_current_frame.png" and
"dataset\grey\clear\<id>_clear_diff_frame.png" or "dataset\grey\drone\<id>_drone_current_frame.png" and
"dataset\grey\drone\<id>_drone_diff_frame.png", where "<id>" is a numeric id of the file pair.
The images are .PNG files with a resolution of 160x120.
The image files are in folders and with file names containing "drone" or "clear" depending if we have predetermined if they contain a drone or not.

The output files shall be stored as "dataset\labels\<id>.txt" using YOLO format, where <id> is the id of the file pair.
<label> <x> <y> <width> <height>
where <label> is 1 for drone and 0 for non-drone. For non-drones, the file should contain
0 0 0 0 0
For drones, label should be 1, x and y normalized x and y coordinates of the drone center position, and width and height normalized width and height of the drone bounding box, where x, y, width, height are normalized 0 to 1 of the width and height of the image.  
The program should have an input field for the dataset root folder.
The program should then scan through all the folders and show two clickable lists of ids and a status if the corresponding image pair is unlabeled or labeled. A file pair is considered "labeled" if there exists a label file. One list of ids which are drones, and one list which are non-drones (or "clear"). It shall list an asterisk next to the number for unlabelled file pairs.
There shall be a button "label clear files" which loops through all the unlabeled clear pairs, and creates label files for them.
There shall be two images showing the two images of the image pair if we click an id. The images should be upscaled to 2x; 320x240. Under the two images it shall show the data of the label file (if any) with drone/nodrone check box, and input fields for x, y, width, height, and a "save" button.
If the pair is classified as drone, the two images shall have a translucent green rectangle shown over the two images, highlighting the area where the drone is marked.
It shall be possible to click and drag in the images to resize the drone label box, updating the numbers, so we can drag and change the box, view the result and then click Save to store the updated label file.
There shall be a "Rescan" button to re-read all the data from disk.
The code should be easy to maintain - we would probably like to tweak it manually afterwards.

The GUI should then show:
Input field with path to the root dataset folder.
A scrollable, clickable list of all found "clear" ids (with an asterisk for unlabelled pairs).
Two image placeholders, resolution 320x240 each.
A scrollable, clickable list of all found "drone" ids (with an asterisk for unlabelled pairs).
Fields showing data for the currently clicked ids:
Labels with paths to the two image files
Checkbox for drone or not
Input boxes for x, y, width, height
Buttons "Save" or "Cancel" ("Cancel" reverting to the stored label file data, if any - and "Save" updating the label file).
Button "Rescan" finding all the file pairs and label data files.
Button "Label clear files" which creates label files for all clear file pairs that do not have a label file.'
